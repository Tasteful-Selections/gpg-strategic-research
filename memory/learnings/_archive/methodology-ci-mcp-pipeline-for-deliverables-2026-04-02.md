# CI + MCP Pipeline for Intelligence Products

## Context

Built the first GroundTruth Market Brief entirely from CI weekly signals (supply squeeze, water constraints, competitive moves) + MCP public data (USDA MARS pricing, NASS production, Census demographics, FRED CPI). No internal GPG data or Nielsen syndicated data required.

## The Learning

The three-repo intelligence pipeline (competitive-intelligence → gpg-strategic-research MCP tools → deliverable) is production-capable today:

1. **CI signals provide the "why"** — what's happening in the market and what it means (supply squeeze, Arable consolidation, Walmart produce revamp)
2. **MCP data provides the "how much"** — quantified evidence (257M lbs imports, 64% Mexico share, 61% purchase frequency)
3. **Internal data will add the "only we know"** — proprietary operational intelligence (when connected via Fabric)
4. **Syndicated data will add the "vs. market"** — competitive benchmarking (Nielsen/Circana, live but GenAI-restricted)

**Key insight:** Each layer makes the preceding layers more valuable. Public data is commodity — anyone could look it up (but most don't). CI adds GPG-specific interpretation. Internal adds proprietary ground truth. Syndicated adds competitive context. Don't wait for all layers to start shipping products.

**MCP sources used in the carrot brief:** USDA MARS (pricing, movement), USDA NASS (production, acreage), FRED (CPI context), Census (demographics — available for future Opportunity Maps), FoodData Central (nutritional comparison).

## When to Apply

Any new GroundTruth product or category brief. Start with CI + MCP, layer in internal and syndicated as available. The template works cross-category — potato, organic veg, and other briefs should follow the same pipeline.

## Confidence

HIGH — demonstrated in working prototype. The carrot brief pulled live data across 4+ MCP sources and wove in CI signals successfully.
